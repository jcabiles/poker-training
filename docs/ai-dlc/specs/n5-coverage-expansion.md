# N5 — grade every action: coverage expansion (delta spec)

Roadmap: `docs/ai-dlc/roadmap/simulate-table.md` Epic 3 slice N5. Contracts: `docs/ai-dlc/contracts/n5-coverage-gaps.md`. Interview locked 2026-07-18: base pack + **defense-range fills** + **multiway grading**; success = fixed-seed non-regression + enumerated newly-mapping families (NO numeric quota). Research inputs (3 parallel Opus agents, 2026-07-18): blind-defense range proposals (copy-ready), multiway strategy-math audit, in-repo multiway line census (6k+ decisions probed).

## Goal (one line)
Raise the share of live Simulate decisions that get an honest grade: align the flop open gate, fill the 9 missing blind-defense ranges, and open a narrowly-scoped 3-way multiway grading path — proving coverage went up on a fixed seed, never silently mis-grading what stays out.

## Research synthesis — resolved tensions (the why behind the scope)
1. **Hero-closes-action vs census volume.** Theory: calling with a live player still to act behind is different math the graders can't see; honest options are gate-off or new data-model work. Census: the #1 multiway cell is hero-as-cold-caller facing a bet, where hero often closes anyway (bet + call(s) in front). RESOLUTION: grade multiway **facing-bet nodes only when hero closes the action** (everyone else already responded to the bet); non-closing spots stay "no baseline yet". **BUILD-PHASE CORRECTION (engine-verified, supersedes the spec-refuter's 21/28 claim): in every 3-way SRP postflop order the BB responds LAST to a bet — the cold-caller NEVER closes. Part C therefore ships as hero = BB DEFENDER (closing) instead of hero = cold-caller; same graders, same machinery, the census volume reads onto the BB-defense cells.**
2. **Limped multiway (45% of MW volume).** No limper ranges authored, no grader family for limped-pot postflop (RES-C §12.5). RESOLUTION: **out** — MW scope is single-raised pots only. Accepted volume loss, honest.
3. **Cold-caller range: loose-capped population model vs existing authored content.** Theory recommends a wide-but-capped live range; census found `VS_RFI`+`call` content already exists (9 pairings). RESOLUTION: **reuse `VS_RFI`+`call`** — consistent with the app-wide convention (villains get the authored "correct" ranges, e.g. villain BB gets the authored defense range), zero new content, zero new NodeContext. Annotate the loose-capped caveat for L1 (persona-aware) to revisit.
4. **"Checked-to" cold-caller cells (census #4).** Hero betting as a NON-aggressor (probe/float) has NO grader family (RES-C §12.2) — census only checked mapper blockers. RESOLUTION: **out** (RES-D territory); only FACING-BET multiway cells ship, which land on the existing `grade_vs_cbet`/`grade_vs_turn_bet`/`grade_vs_river_bet`.
5. **Donk-leads.** Census shows the multiway bettor is often the BB, not the raiser — but the facing graders model facing the AGGRESSOR's range; no donk-range content or grader exists (RES-C §12.1). RESOLUTION: multiway gate requires **the bettor = the preflop raiser** (c-bet shape), canonical size (`_is_canonical_bet` + `_CANON_BET_TOL`).
6. **4+-way.** Theory: published research quantifies HU-vs-multiway only; the existing merit knobs are tuned to ~3-way and under-adjust 4-way. RESOLUTION: **3-way only**; 4+ stays "no baseline yet".
7. **`_apply_multiway` audit verdict: keep** (directions and rough magnitudes research-backed, RES-C-blessed). Two fixes ship BEFORE the mappers make it live-reachable: (a) extend the aggressor-side dampen to thin `weak_made` value ("thin value disappears multiway" is the most-cited result the knobs miss); (b) comment-annotate `_villain_pos`/`range_advantage`/`_cbet_fold_equity_value` as HU-shaped inputs deliberately corrected downstream (fold equity should compound multiplicatively across defenders; the 0.6 dampen is the honest patch).

## In scope

### Part A — base pack (roadmap-mandated)
- **A1 flop-gate alignment**: `map_flop_cbet`'s exact-open match (`grade_map_postflop.py:71`, `_EPS` vs the exact per-seat canonical) → the same `[2.0, _STD_OPEN_CAP]` band `_hu_srp_preflop` uses (:203-204). **Refuter HIGH: relaxing the gate alone is NOT enough** — the downstream pot check (`abs(pot - (2*osize+0.5))`, :85) and small/big bet-fraction math key on the CANONICAL `_OPEN_SIZE`, so a non-canonical in-band open would be re-rejected (or mis-priced). A1 = relax the gate AND switch the pot/bet math to `raises[0].amount_bb` (the ACTUAL live open), mirroring how `_hu_srp_preflop` returns `open_to` for downstream math. Belt test: a 2.5bb open from a 3.0-canonical seat maps with pot/SPR/bet sizes derived from 2.5, not 3.0.
- **A2 belt test**: bot-driven test proving (i) `map_flop_cbet` maps a 3.0 open, (ii) `map_vs_turn_bet`/`map_vs_river_bet` fire against a bot-rounded `round(f*pot, 2)` bet (locks the N4b `_CANON_BET_TOL` fix at the integration level).
- **A3 spot-dims persistence**: additive nullable columns on `sim_decision` (+ Alembic migration 0012) storing the graded spot's dims for N1's by-spot drill-down: `node_context` (str), `street` exists already — add `position` (hero), `facing_position`, `players_in_pot` (int). **Refuter HIGH — precise wiring:** `_sim_decision_row` (`sim_session.py:241`) gains a `spot: Spot | None` param (the call site at :733 has it in scope; `EvaluationResult` carries none of these dims). None-spot (unmappable) semantics: `position` ALWAYS populated from `state.seats[HERO_SEAT].position`; `facing_position`/`players_in_pot`/`node_context` null unless `spot is not None`. No FE (read-model comes later).
- **A4 fixed-seed coverage baseline**: a deterministic harness (fixed seed + fixed lineup, N hands, scripted hero policy) that reports graded/total decisions; record the pre-N5 number in the slice notes; test asserts post-N5 ≥ pre-N5.

### Part C — multiway grading, narrowly scoped (the "minimum honest model")
**The ONE new multiway family: hero = preflop COLD-CALLER in a 3-way single-raised pot, facing the RAISER's canonical c-bet/barrel, hero CLOSES the action — flop, turn, river.** (Census: the dominant multiway cells by 3×.)
- **C1 N-way preflop gate**: `_mw_srp_preflop(state)` — one non-blind opener at an in-band open, exactly ONE non-blind cold-caller (hero), BB called or folded... scope precisely: live players = {opener, hero-caller, BB} (3-way) with BB having called the open; all IN (no ALLIN). Returns roles. (Generalizes `_hu_srp_preflop`'s single-caller rule; does NOT require hero=BB.)
- **C2 multiway street gate**: per street, accepts sequences where the RAISER bets a canonical size and every other live defender has already responded (called or folded) before hero — i.e., hero closes. Rejects donk-leads (bettor ≠ raiser), raises in front, and any other order → None.
- **C3 mappers**: `map_mw_flop_vs_cbet`, `map_mw_vs_turn_bet`, `map_mw_vs_river_bet` building 3-player Spots via a generalized `_faced_bet_spot` (players list carries all three; `players_in_pot=3` ⇒ `is_multiway` True ⇒ `_apply_multiway` engages; N4b two-RAISE legs + `FACING_RAISE_MULTS` reused as-is). Prior-street re-verification mirrors the HU turn/river discipline (canonical prior bets, both defenders called).
- **C4 ranges**: hero (caller) = `VS_RFI`+`call` for (hero_pos, opener_pos); villain aggressor = `RFI`+`raise`. **Refuter HIGH — the BB's range has NO grading consumer** (`grade_vs_cbet` takes ONE `villain_range`; `PlayerState` has no range field; `_apply_multiway` reads merits+category only): the BB's blind-defense entry is used ONLY as a mapping GATE (entry must exist for the pairing or → None — keeps the "never fabricate ranges" discipline and leans on Part B's fills), NOT as a grader input. Stated plainly so nobody wires phantom plumbing. Missing pairing (either range or the BB gate) → None (honest).
- **C5 grader-side fixes (ship first)**: `_apply_multiway` gains an aggressor-side `weak_made` dampen — **new named constant `_MW_THIN_VALUE_DAMPEN = 0.7`** applied to the small/big bet merits when `cat_effective == "weak_made"` and not facing (refuter MED: concrete knob, sits between the bluff dampen 0.6 and neutral — "thin value disappears multiway" is the most-cited research result the current knobs miss; genuinely new branch, no pinned test exists). Plus comment annotations on `_villain_pos`/`range_advantage`/`_cbet_fold_equity_value`: HU-shaped inputs deliberately corrected downstream (fold equity should compound multiplicatively across defenders; the dampens are the honest patch). S8's MW unit tests extend for the new branch. **Blast-radius note (refuter probe): verify whether Practice/drill can serve MW spots today (`build_*_spot(players_in_pot=...)` callers) — if any live caller passes >2, C5 changes those Practice grades; enumerate + accept explicitly in the ticket, don't assume zero callers.**
- **C6 dispatcher + display**: `grade_map.py` tries MW mappers after the HU chain (disjoint by live-count); `sim_session.py` facing-raise two-size display gate extends to the MW mappers (same read-legs-off-spot parity pattern).
- **C7 signature/persistence**: dormant "mw" append verified live (dry-run: distinct stable hash, HU hashes byte-identical); e2e test = a 3-way hand persists a graded `SimDecision` with `players_in_pot=3` (A3's new dim).
- **Stays "no baseline yet" (enumerated, tested)**: 4+-way · limped pots · donk-leads · hero-doesn't-close facing spots · checked-to caller (probe) · BB-defender multiway (hero rarely closes; low census volume) · multiway 3-bet pots.
